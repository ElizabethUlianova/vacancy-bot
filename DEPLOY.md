# Деплой на GitHub Actions (бесплатно, без сервера)

Бот будет запускаться сам каждые ~15 минут в облаке GitHub. Твой компьютер не нужен.
Токен хранится в защищённых **Secrets**, а не в коде.

## Что понадобится
- Аккаунт на [github.com](https://github.com) (бесплатный).
- Всё в этой папке уже готово к загрузке.

---

## Шаг 1. Создать репозиторий
1. Зайди на github.com → справа вверху **＋** → **New repository**.
2. Имя: например `vacancy-bot`.
3. Выбери **Public** — так минуты Actions бесплатны без лимита.
   (Токена в коде нет, он пойдёт в Secrets, поэтому публичный репозиторий безопасен.
   Если хочешь **Private** — тоже можно, но тогда поставь запуск раз в 30 мин, см. ниже.)
4. **Create repository**.

## Шаг 2. Загрузить файлы
Проще всего через браузер:
1. На странице репозитория → **uploading an existing file**.
2. Перетащи туда содержимое этой папки:
   `vacancy_bot.py`, `config.json`, `requirements.txt`, `README.md`, `.gitignore`
   и папку `.github` (внутри неё `workflows/vacancy-bot.yml`).
   ⚠️ Папку `.github` через drag-and-drop браузер иногда не берёт — если так,
   создай файл вручную: **Add file → Create new file**, впиши путь
   `.github/workflows/vacancy-bot.yml` и вставь его содержимое.
3. **Commit changes**.

## Шаг 3. Добавить Secrets (токен и chat_id)
1. В репозитории: **Settings** → слева **Secrets and variables** → **Actions**.
2. **New repository secret**, создай два:

   | Name | Secret |
   |------|--------|
   | `TELEGRAM_BOT_TOKEN` | `8898483238:AAE_0ghdZWcT80Df_KcmLn7edZ_el2-4_Xo` |
   | `TELEGRAM_CHAT_ID`   | `160214155` |

3. Сохрани оба.

## Шаг 4. Включить и проверить
1. Вкладка **Actions** → если попросит — **I understand my workflows, enable them**.
2. Слева выбери **vacancy-bot** → справа **Run workflow** (кнопка ручного запуска).
3. Первый запуск просто запомнит текущие вакансии (без сообщений) и создаст `seen.json`.
4. Дальше каждые ~15 минут бот будет присылать в Telegram только новые вакансии.

Готово. Больше ничего делать не нужно — бот работает сам.

---

## Полезно знать

**Как поменять ключевые слова или источники.** Отредактируй `config.json` прямо на
GitHub (открой файл → карандаш ✏️ → Commit). Изменения подхватятся со следующего запуска.

**Задержки.** GitHub в час пик может запускать по расписанию с опозданием на 10–30 мин —
это нормально для бесплатного плана. «Как только появится» = «в течение ~15–40 мин».

**Приватный репозиторий.** Бесплатно даётся 2000 минут/мес. При запуске каждые 15 мин
можно выйти за лимит. Если репозиторий приватный — открой `.github/workflows/vacancy-bot.yml`
и поменяй `*/15` на `*/30` (раз в 30 мин, ~1440 мин/мес, укладывается).

**Автоотключение.** GitHub выключает расписание после 60 дней без активности в репозитории.
Достаточно раз в пару месяцев зайти и нажать **Run workflow**, чтобы сбросить счётчик.

**Логи.** Вкладка **Actions** → любой запуск → шаг *Run vacancy bot* показывает,
сколько вакансий найдено и отправлено.
